# AGENTS.md

Context for AI coding agents (Claude Code, Codex, Cursor, etc.) working on this repo. Humans welcome too.

## What this is

Personal site for Martin Miglio. Single app, not a monorepo. React 19 + TanStack Start, rendered server-side on AWS Lambda via SST, fronted by CloudFront.

## Stack

- **Runtime:** Node 22, package manager is `bun@1.2.17` (see `packageManager` in `package.json`)
- **App:** React 19, TanStack Start + Router, Vite 7
- **Styling:** Tailwind 4, shadcn-ui (Radix primitives under `src/components/`)
- **Infra:** SST v4 → AWS Lambda (ARM64) + CloudFront. See `infra/AGENTS.md`.
- **Tooling:** oxlint + oxfmt (not biome/prettier/eslint), lefthook (not husky), vitest

## Dev loop

```bash
bun install              # also installs git hooks via the `prepare` script
bun run dev:mono         # sst dev --mode mono (full stack locally)
bun run dev              # vite only, if you don't need sst
bun run check            # oxfmt --check && oxlint
bun run typecheck        # tsc --noEmit
bun run test             # vitest
```

Fixers: `bun run format` (oxfmt), `bun run lint:fix` (oxlint --fix).

## Conventions

- **Use `bun`, not `npm`/`pnpm`/`yarn`.** `bunx` over `npx`.
- **Don't reach for biome, prettier, or eslint.** This repo uses oxlint + oxfmt. Config: `.oxlintrc.json`, `.oxfmtrc.json`.
- **Git hooks are lefthook**, config in `lefthook.yml`. Pre-commit runs oxfmt + oxlint on staged files and typechecks if `.ts`/`.tsx` changed. Pre-push runs the full check + typecheck. Don't bypass with `--no-verify` unless explicitly asked.
- **No commit co-authors.** Don't add Claude/Anthropic or other agent authorship trailers.

## Layout

```
src/          app code — components, pages, routes, hooks, lib, data, assets
  routeTree.gen.ts   generated by @tanstack/router-plugin, don't edit by hand
infra/        SST config + one-time OIDC bootstrap (see infra/AGENTS.md)
.github/workflows/   4 workflows: check, deploy, deploy-preview, destroy-preview
sst.config.ts        main SST app definition (imported by `sst` CLI)
```

## Deploys

All deploys go through GitHub Actions via OIDC — never `sst deploy` from a laptop to `production` or `develop`. For deploy internals, stage→domain mapping, and the OIDC bootstrap, see [`infra/AGENTS.md`](./infra/AGENTS.md).
